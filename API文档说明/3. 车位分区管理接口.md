### 3. 车位分区管理接口

#### 3.1 获取所有车位分区列表
**接口地址**: `GET /api/parking-zone/list`

**请求头**:
```
Authorization: Bearer {token}
```

**请求参数**: 无

**响应示例**:
```json
{
  "code": 200,
  "status": true,
  "message": "操作成功",
  "data": [
      {
          "id": 11,
          "parkAreaId": 1,
          "zoneId": 4,
          "spaceNumber": "B-001",
          "spaceType": 1,
          "status": 1,
          "latitude": 25.257826,
          "longitude": 110.122571,
          "bindUserId": null,
          "createTime": "2026-02-01T00:00:00",
          "updateTime": "2026-02-26T01:36:27",
          "currentOccupiedStatus": 1
      }
  ],
  "timestamp": 1777482951451
}
```

#### 3.2 根据ID获取车位分区详情
**接口地址**: `GET /api/parking-zone/{id}`

**请求头**:
```
Authorization: Bearer {token}
```

**请求参数**:
- `id`: 分区ID（路径参数）

**响应示例**:
```json
{
    "code": 200,
    "status": true,
    "message": "操作成功",
    "data": {
        "id": 3,
        "parkAreaName": "园区A",
        "zoneName": "A区",
        "description": "A去车位",
        "sortOrder": 1,
        "status": 1,
        "createTime": "2026-02-01T00:00:00",
        "updateTime": "2026-04-28T00:32:37",
        "park_area_id": 1
    },
    "timestamp": 1777371238348
}
```

#### 3.3 创建车位分区
**接口地址**: `POST /api/parking-zone`

**请求头**:
```
Authorization: Bearer {token}
```

**请求参数**:
```json
{
  "park_area_id": 2,
  "zoneName": "B区",
  "description": "B区车位",
  "sortOrder": 2,
  "status": 1
}
```

**响应示例**:
```json
{
    "code": 200,
    "status": true,
    "message": "操作成功",
    "data": {
        "id": 6,
        "parkAreaName": "园区B",
        "zoneName": "B区",
        "description": "B区车位",
        "sortOrder": 2,
        "status": 1,
        "createTime": "2026-04-28T18:17:59",
        "updateTime": "2026-04-28T18:17:59",
        "park_area_id": 2
    },
    "timestamp": 1777371479298
}
```

**注意事项**:
- 同一个园区内分区名称必须唯一
- 如果园区已存在相同名称的分区，会返回错误（RC1301）

#### 3.9 批量更新分区状态
**接口地址**: `PUT /api/parking-zone/batch/status?ids=3,4,5&status=0`

**请求头**:
```
Authorization: Bearer {token}
```


**请求参数**:
- `ids`: 分区ID列表，用逗号分隔
- `status`: 状态（0-禁用，1-启用）

**响应示例**:
```json
{
    "code": 200,
    "status": true,
    "message": "操作成功",
    "data": true,
    "timestamp": 1777372312089
}data是ture后面可能会改，++可以不需要
```
